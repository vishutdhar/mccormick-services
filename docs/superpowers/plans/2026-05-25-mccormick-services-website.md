# McCormick Services Website Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:subagent-driven-development` (recommended) or `superpowers:executing-plans` to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build and deploy a one-page Next.js marketing site for McCormick Services (Mike McCormick, Macomb County MI) that drives phone calls and quote requests for power washing, painting, lawn care, and fall cleaning.

**Architecture:** Next.js App Router single-page site rendered at build time (SSG), with one client-side interactive island (the quote form). Styled with Tailwind v4 + shadcn/ui. Hosted on Vercel free tier with `mccormick-services.vercel.app` as the initial public URL. Quote form submissions go to Formspree (free tier) which emails Mike directly. SEO metadata and `LocalBusiness` JSON-LD baked at build time.

**Tech Stack:** Next.js (latest, App Router) · TypeScript · Tailwind CSS v4 · shadcn/ui · Lucide icons · next/font (Manrope + Inter) · Formspree · Vitest + Testing Library · Vercel.

**Source-of-truth spec:** `docs/superpowers/specs/2026-05-25-mccormick-services-design.md`

**Design rationale**

- Single-page composition in `app/page.tsx` keeps the route static. Each section is its own component for isolation and easy reorder/removal.
- `lib/business.ts` is the single source of truth for everything Mike might update later: phones, services, cities, hours, owner name. Editing one file updates the whole site.
- `lib/schema.ts` consumes `business.ts` and emits valid `LocalBusiness` JSON-LD — no duplication between human-readable copy and structured data.
- `quote-form.tsx` is the only client component (`"use client"`), so the rest of the page ships as zero-JS HTML.

const digitsOnly = (phone: string): string => phone.replace(/\D+/g, "");

export function formatTelHref(phone: string): string {
  const digits = digitsOnly(phone);
  if (digits.length === 10) return `tel:+1${digits}`;
  if (digits.length === 11 && digits.startsWith("1")) return `tel:+${digits}`;
  throw new Error(`formatTelHref: unsupported phone number: ${phone}`);
}

export function formatPhoneDisplay(phone: string): string {
  const digits = digitsOnly(phone);
  const ten = digits.length === 11 && digits.startsWith("1") ? digits.slice(1) : digits;
  if (ten.length !== 10) throw new Error(`formatPhoneDisplay: expected 10 digits, got ${phone}`);
  return `(${ten.slice(0, 3)}) ${ten.slice(3, 6)}-${ten.slice(6)}`;
}

"use client";

import { useState, type FormEvent } from "react";
import { SERVICES } from "@/lib/business";

type Status = "idle" | "submitting" | "success" | "error";

export function QuoteForm({ endpoint }: { endpoint: string }) {
  const [status, setStatus] = useState<Status>("idle");

  async function onSubmit(event: FormEvent<HTMLFormElement>) {
    event.preventDefault();
    const form = event.currentTarget;
    const data = new FormData(form);

    // Honeypot: bots fill hidden fields; humans don't.
    if ((data.get("company") as string)?.length) {
      setStatus("success");
      form.reset();
      return;
    }

    setStatus("submitting");
    try {
      const res = await fetch(endpoint, {
        method: "POST",
        body: data,
        headers: { Accept: "application/json" },
      });
      if (!res.ok) throw new Error(`Formspree ${res.status}`);
      setStatus("success");
      form.reset();
    } catch {
      setStatus("error");
    }
  }

  if (status === "success") {
    return (
      <p className="rounded-md bg-brand-lime-soft text-brand-forest p-4 font-display text-lg">
        Thanks — we'll be in touch shortly.
      </p>
    );
  }

  return (
    <form onSubmit={onSubmit} className="space-y-4">
      <div>
        <label htmlFor="name" className="block font-display text-brand-forest mb-1">
          Name
        </label>
        <input
          id="name"
          name="name"
          type="text"
          required
          autoComplete="name"
          className="w-full rounded-md border border-brand-olive/40 bg-white px-3 py-2"
        />
      </div>

      <div>
        <label htmlFor="phone" className="block font-display text-brand-forest mb-1">
          Phone
        </label>
        <input
          id="phone"
          name="phone"
          type="tel"
          required
          autoComplete="tel"
          inputMode="tel"
          className="w-full rounded-md border border-brand-olive/40 bg-white px-3 py-2"
        />
      </div>

      <div>
        <label htmlFor="email" className="block font-display text-brand-forest mb-1">
          Email (optional)
        </label>
        <input
          id="email"
          name="email"
          type="email"
          autoComplete="email"
          className="w-full rounded-md border border-brand-olive/40 bg-white px-3 py-2"
        />
      </div>

      <div>
        <label htmlFor="service" className="block font-display text-brand-forest mb-1">
          Service
        </label>
        <select
          id="service"
          name="service"
          required
          defaultValue=""
          className="w-full rounded-md border border-brand-olive/40 bg-white px-3 py-2"
        >
          <option value="" disabled>
            Choose a service…
          </option>
          {SERVICES.map((s) => (
            <option key={s.slug} value={s.slug}>
              {s.title}
            </option>
          ))}
          <option value="other">Other</option>
        </select>
      </div>

      <div>
        <label htmlFor="message" className="block font-display text-brand-forest mb-1">
          Message (optional)
        </label>
        <textarea
          id="message"
          name="message"
          rows={4}
          className="w-full rounded-md border border-brand-olive/40 bg-white px-3 py-2"
        />
      </div>

      {/* Honeypot. Hidden from real users; bots fill anything labeled "company". */}
      <div className="hidden" aria-hidden="true">
        <label htmlFor="company">Company</label>

- [ ] **Step 3: Implement `src/lib/schema.ts`**

```ts
import { BUSINESS, PRIMARY_PHONE, SECONDARY_PHONE, SERVICES, formatTelHref } from "./business";

type SchemaOffer = {
  "@type": "Offer";
  itemOffered: { "@type": "Service"; name: string; description: string };
};

type SchemaHours = {
  "@type": "OpeningHoursSpecification";
  dayOfWeek: string;
  opens: string;
  closes: string;
};

export type LocalBusinessSchema = {
  "@context": "https://schema.org";
  "@type": "LocalBusiness";
  name: string;
  description: string;
  telephone: string;
  areaServed: string;
  url: string;
  priceRange: string;
  openingHoursSpecification: SchemaHours[];
  makesOffer: SchemaOffer[];
  contactPoint: {
    "@type": "ContactPoint";
    telephone: string;
    contactType: "customer service";
  }[];
};

const toE164 = (phone: string): string =>
  formatTelHref(phone).replace(/^tel:/, "");

export function buildLocalBusinessSchema(): LocalBusinessSchema {
  return {
    "@context": "https://schema.org",
    "@type": "LocalBusiness",
    name: BUSINESS.name,
    description: BUSINESS.tagline,
    telephone: toE164(PRIMARY_PHONE),
    areaServed: BUSINESS.areaServed,
    url: BUSINESS.url,
    priceRange: BUSINESS.priceRange,
    openingHoursSpecification: BUSINESS.hours.map((h) => ({
      "@type": "OpeningHoursSpecification",
      dayOfWeek: h.day,
      opens: h.open,
      closes: h.close,
    })),
    makesOffer: SERVICES.map((s) => ({
      "@type": "Offer",
      itemOffered: { "@type": "Service", name: s.title, description: s.description },
    })),
    contactPoint: [
      {
        "@type": "ContactPoint",
        telephone: toE164(PRIMARY_PHONE),
        contactType: "customer service",
      },
      {
        "@type": "ContactPoint",
        telephone: toE164(SECONDARY_PHONE),
        contactType: "customer service",
      },
    ],
  };
}

// Safety: escapes `<` so the JSON payload cannot break out of its enclosing
// <script type="application/ld+json"> element. All inputs to the schema are
// compile-time constants from lib/business.ts, but the escape is cheap and
// defensive.
export function serializeSchemaForScript(schema: LocalBusinessSchema): string {
  return JSON.stringify(schema).replace(/</g, "\\u003c");
}
```

- [ ] **Step 5: Inject the JSON-LD in `src/app/layout.tsx`**

Follow the canonical Next.js JSON-LD pattern documented at <https://nextjs.org/docs/app/guides/json-ld>. Inside `<body>`, before `{children}`, render a `<script type="application/ld+json">` element. Use React's inline-HTML escape-hatch attribute to set the script's contents to `serializeSchemaForScript(buildLocalBusinessSchema())`.

Notes for the implementer:
- Import: `import { buildLocalBusinessSchema, serializeSchemaForScript } from "@/lib/schema";`
- The escape-hatch attribute on a React `<script>` element is a `__html`-keyed object passed to the inline-HTML prop documented in Next.js's JSON-LD guide; suppress the corresponding ESLint warning on that single line.
- The output of `serializeSchemaForScript` is already `<`-escaped, so the injected content cannot break the script element.
